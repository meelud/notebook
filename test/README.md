# تست‌ها — شبکه‌ی ایمنی

این تست‌ها **رفتار فعلیِ منطق موسیقی/احساسِ پروژه رو قفل می‌کنن** تا هر تغییرِ بعدی بدون لرزه انجام شه: لحظه‌ای که یه ویرایش، خروجیِ `detectMood`/`analyzeText` یا محاسباتِ گام/آکورد رو عوض کنه، تست داد می‌زنه.

- **صفر dependency** — با تست‌رانرِ نیتیوِ Node اجرا می‌شه (`node --test`).
- فقط توابعِ **خالصِ** `js/mood.js` رو تست می‌کنه (که به Web Audio وابسته نیستن).

## اجرا

```sh
npm test
```

اگه چیزی fail شد، یعنی خروجی نسبت به اسنپ‌شاتِ قفل‌شده عوض شده.

- **اگه تغییر ناخواسته بود** → کدت یه چیزی رو شکسته، برگرد عقب.
- **اگه تغییر عمدی و درست بود** → اسنپ‌شات رو به‌روز کن:

```sh
npm run test:update
```

## فایل‌ها

```
test/
  mood.snapshot.test.mjs      – خودِ تست‌ها + نمونه‌متن‌ها
  __snapshots__/mood.snap.json – خروجیِ قفل‌شده (این رو دستی ویرایش نکن)
```
